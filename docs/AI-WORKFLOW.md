# AI 开发工作流 · AI-WORKFLOW

> 这个 demo 本身就是用 AI 协作（Vibe Coding）做出来的。这里记录把一个模糊的玩法点子，**拆解成 AI 能落地的描述、再快速验证**的过程——也就是岗位说的「把需求转化为 AI 可理解的描述」「AI 工作流探索」。

## 0. 一句话方法论

> **先定「判定」，再写「表演」；数据先行，prompt 即内容，浏览器即测试。**

## 1. 把玩法拆成 AI 能执行的层

一开始的点子只有一句话：「做一个审讯 AI 嫌疑人的推理游戏」。直接丢给模型「帮我写这个游戏」只会得到一坨能跑但不好玩、也不可控的东西。于是先把玩法**拆层**：

| 层 | 谁负责 | 拆解产物 |
| --- | --- | --- |
| 胜负判定 | **代码（确定性）** | `evidence → lie` 映射、`requiredBreaks` 阈值 |
| 角色表演 | **LLM（实时）** | 每个嫌疑人的 system prompt |
| 离线兜底 | 作者剧本 | 关键词 → 台词 |
| 演出与流程 | Phaser 场景 | 立绘动画、「破绽」震屏、场景切换 |

**最关键的判断在第一行**：先决定「什么绝不能交给 LLM」（胜负），剩下的才放心交给它。这是整个项目可控、可玩的前提。

## 2. 数据模型先行

在写任何画面之前，先和 AI 一起敲定 `CaseData` 数据结构（[`src/data/types.ts`](../src/data/types.ts)）：线索、证物、嫌疑人、`lieBreaks`、`requiredBreaks`。数据结构一旦清晰，后面「加内容」和「写引擎」就解耦了——AI 写引擎，人写/审内容。

## 3. Prompt 即内容（最花心思的一步）

把每个嫌疑人当成一份**规格说明**来写，而不是随手一句「你是个嫌疑人」。固定骨架（身份 / 绝密真相 / 谎言 / 触发条件 / 表演方式 + 护栏）见 [PROMPT-DESIGN](PROMPT-DESIGN.md)。

迭代时反复问自己 / 让 AI 自检：

- 直接问「是不是你干的」会不会秒供？
- 三条谎言的「反证」是否都和某件**具体证物**一一对应？
- 无辜者会不会被冤枉？会不会乱咬别人？
- 离线剧本能否覆盖到「能把案子查完」的最小话题集？

## 4. 用「离线模式」当测试夹具

实时 LLM 有随机性，不适合当回归测试。于是离线 `ScriptedProvider` 一举两得：

- 对玩家：**无需 Key 即可完整试玩**（招聘官打开就能玩）；
- 对开发：一个**确定性**的对话替身，可在浏览器里把「简报 → 审讯 → 出示证据 → 击破 → 指认 → 结案」整条主线点一遍，验证判定、UI、状态流转——全程零随机。

实际验证就是这么做的：本地起服务，从主菜单一路点到结案，确认 `击破谎言 2/2 → 定罪成功`，控制台零报错。

## 5. 工具链

- **Claude Code / Cursor**：成对编程，生成 + 重构 + 自检；
- **Vite + TypeScript**：类型即契约，让 AI 改代码时不容易跑偏；
- **Phaser 3**：场景与演出；
- **OpenAI 兼容 BYOK**：一套接口接 GPT / DeepSeek / Kimi / 本地模型，零后端。

## 6. 复盘 · 下一步

**做对的**：先锁死判定边界，再放手让 LLM 演——可控性与「活人感」兼得。
**可改进**：

- 给嫌疑人加「信任/情绪」状态，让累计对话影响演出（但仍不影响判定）；
- 多案章节化、对话中解锁隐藏证物；
- 加一个可选的极简代理，绕开个别供应商的 CORS 限制；
- 对实时回复做轻量「人设守卫」二次校验（防越狱泄底）。
